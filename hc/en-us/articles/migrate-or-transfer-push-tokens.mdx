# Migrate or Transfer Push Tokens

When migrating to MoEngage from another platform, you may need to transfer your existing push tokens to maintain user engagement without interruption.

## Understanding Push Tokens

### What are Push Tokens?
Push tokens are unique identifiers that allow platforms to send push notifications to specific devices. They are:
- **Device-Specific**: Each device has a unique token
- **Platform-Specific**: Different for iOS and Android
- **App-Specific**: Tied to your specific app
- **Essential for Notifications**: Required for sending push notifications

### Why Transfer Tokens?
- **Maintain Engagement**: Keep users engaged during migration
- **Avoid Re-opt-in**: Users don't need to re-opt-in for notifications
- **Seamless Transition**: Smooth migration experience
- **Preserve User Base**: Maintain your existing user base

## Migration Process

### Step 1: Export Tokens from Current Platform
1. **Access Current Platform**: Log into your current push notification platform
2. **Export Tokens**: Export all push tokens and associated user data
3. **Format Data**: Ensure data is in the correct format for MoEngage
4. **Verify Data**: Check that all tokens are properly exported

### Step 2: Prepare Data for MoEngage
1. **Data Format**: Ensure data matches MoEngage requirements
2. **User Mapping**: Map users to their push tokens
3. **Data Validation**: Validate token format and user data
4. **Backup Data**: Keep a backup of the original data

### Step 3: Import to MoEngage
1. **Access MoEngage**: Log into your MoEngage dashboard
2. **Go to Imports**: Navigate to the data import section
3. **Upload Data**: Upload your push token data
4. **Verify Import**: Check that tokens were imported correctly

## Data Format Requirements

### Required Fields
- **User ID**: Unique identifier for each user
- **Push Token**: The actual push token
- **Platform**: iOS or Android
- **App Version**: Version of the app
- **Last Updated**: When the token was last updated

### Optional Fields
- **User Attributes**: Additional user information
- **Subscription Status**: Whether user is subscribed
- **Opt-in Date**: When user opted in for notifications
- **Device Information**: Device type and model

### File Formats
- **CSV**: Comma-separated values file
- **JSON**: JavaScript Object Notation
- **Excel**: Microsoft Excel file
- **API**: Direct API integration

## Platform-Specific Considerations

### iOS Push Tokens
- **APNs Tokens**: Apple Push Notification service tokens
- **Token Format**: 64-character hexadecimal string
- **Environment**: Development or production environment
- **Certificate**: Ensure proper certificate configuration

### Android Push Tokens
- **FCM Tokens**: Firebase Cloud Messaging tokens
- **Token Format**: Variable length string
- **Project ID**: Firebase project identifier
- **Server Key**: Firebase server key configuration
