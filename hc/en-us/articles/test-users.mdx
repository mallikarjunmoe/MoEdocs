# Test Users

Test users are essential for testing your MoEngage integration, campaigns, and features before deploying them to your live users. This guide will help you create and manage test users effectively.

## Overview

### What are Test Users?
Test users are:
- **Development Accounts**: Accounts used for testing and development
- **Isolated Environment**: Separate from your live user base
- **Controlled Testing**: Allow you to test features safely
- **Quality Assurance**: Help ensure quality before going live

### Why Use Test Users?
- **Safe Testing**: Test features without affecting real users
- **Quality Assurance**: Ensure features work correctly
- **Development**: Develop and debug features
- **Training**: Train team members on new features

## Creating Test Users

### Method 1: Through MoEngage Dashboard
1. **Log In**: Log into your MoEngage dashboard
2. **Go to Users**: Navigate to the Users section
3. **Create User**: Click "Create User" or "Add User"
4. **Fill Details**: Enter user details and mark as test user
5. **Save**: Save the test user

### Method 2: Through SDK
1. **Use SDK**: Use the MoEngage SDK to create test users
2. **Set Test Flag**: Set the test user flag when creating users
3. **Verify**: Verify that test users are properly created
4. **Test**: Test the integration with test users

### Method 3: Through API
1. **Use API**: Use the MoEngage API to create test users
2. **Set Parameters**: Set appropriate parameters for test users
3. **Validate**: Validate that test users are created correctly
4. **Monitor**: Monitor test user activity
